# AI情绪花园（Mood Garden）架构设计 V1.0

> 配套文档：[需求文档.md](需求文档.md)
> 技术栈：React 18 + TypeScript + Vite + TailwindCSS + Framer Motion + Three.js
> 存储：LocalStorage（比赛版无后端）

---

## 一、技术架构

### 分层模型

应用按「展示层 → 逻辑层 → 数据层 → 引擎层」四层组织，单向依赖下层。

```
┌─────────────────────────────────────────────┐
│  展示层 Pages / Components（只渲染，无逻辑）      │
├─────────────────────────────────────────────┤
│  逻辑层 Hooks / Context（状态 + 业务编排）        │
├─────────────────────────────────────────────┤
│  数据层 Services / Storage（持久化 + AI 调用）     │
├─────────────────────────────────────────────┤
│  引擎层 Garden Engine（Three.js 渲染）           │
└─────────────────────────────────────────────┘
```

### 关键原则

- **引擎与 React 解耦**：Three.js 场景封装在 `garden/`，React 仅通过 `useGardenScene` 与之通信。
- **AI 可降级**：所有 AI 调用走 `services/ai/`，提供 mock 实现，无 key 也能演示。
- **单一数据源**：八情绪配置集中在 `config/emotions.ts`，新增情绪只改一处。
- **派生而非存储**：成长阶段、花园元素、统计由记录计算得出，避免数据不一致。
- **页面只展示**：业务编排进 hooks，符合 CLAUDE.md 规范。

## 二、页面架构

| 路由 | 页面 | 职责 | 核心 Hook |
|---|---|---|---|
| `/` | HomePage | Logo、标题、入口、入场动画 | — |
| `/record` | RecordPage | 日期 + 情绪选择 + 描述 + 提交 | `useMoodForm` `useMoodRecords` |

页面文件只做布局 + 组合 + 调用 hook，预期 < 120 行。

## 四、数据流设计

```
用户操作（选情绪/写描述/提交）
        │
        ▼
  Hook 编排（useMoodForm → useMoodRecords）
        │
        ├──► services/ai      ── 情绪分析 / 陪伴语（可 mock 降级）
        ├──► services/storage ── 写入 LocalStorage（mood_records）
        ▼
  全局状态 GardenContext（当天情绪 + 历史记录 + 成长阶段）
        │
        ├──► GardenPage ──► useGardenScene ──► garden 引擎
        └──► TimelinePage / 统计（订阅同一份记录）
```

### 状态管理策略

- **全局**：`GardenContext`（Context + useReducer），唯一真相源。
- **局部**：表单态、弹窗、筛选用页面内 `useState`。
- **派生**：成长阶段、花园元素、统计全部计算得出。
- **持久化**：`useMoodRecords` 变更同步写 LocalStorage，启动时 hydrate。

## 六、八情绪 → 花园元素映射表

> 单一数据源，对应 `config/emotions.ts`。新增/调整情绪只改此表。

| 情绪 | key | emoji | 主配色 | 植物元素 | 粒子/特效 | 天气/光效 | 昼夜偏好 |
|---|---|---|---|---|---|---|---|
| 开心 | `happy` | 😊 | 暖黄 `#F6C453` | 向日葵 | 金色光点、蝴蝶 | 明亮阳光 | 白天 |
| 平静 | `calm` | 🍃 | 草绿 `#8BC79A` | 白色小花、草地 | 微风粒子 | 柔光 | 白天 |
| 兴奋 | `excited` | ✨ | 彩虹 `#FF7AB6` | 烟花花朵 | 彩虹光效、闪烁粒子 | 高亮 | 黄昏 |
| 焦虑 | `anxious` | 🌧️ | 冷蓝 `#5C8AC2` | 蓝色荆棘 | 雨滴、雾气 | 阴天 | 阴 |
| 迷茫 | `confused` | 🌫️ | 紫雾 `#A78BC2` | 紫色花海 | 漂浮云层 | 朦胧雾光 | 黄昏 |
| 难过 | `sad` | 🌙 | 暗蓝 `#3E4C6B` | 月光花 | 萤火虫 | 月光 | 夜晚 |
| 愤怒 | `angry` | ⚡ | 赤红 `#D9534F` | 红色彼岸花 | 火焰粒子 | 强对比 | 黄昏 |
| 孤独 | `lonely` | ❄️ | 冰青 `#9FC8D6` | 冬雪树 | 冷色光效、飘雪 | 冷光 | 夜晚 |

派生规则：

- **多选情绪**：花园元素叠加渲染；主配色取第一个选中情绪；天气取「强度最高」情绪。
- **配色 token**：主配色进入 `config/theme.ts`，供 Tailwind 与场景共用。

## 七、核心 TypeScript 接口草案

> 对应 `src/types/`，全程禁用 `any`（CLAUDE.md 规范 9）。

### `types/mood.ts`

```typescript
/** 八种情绪枚举键，与 config/emotions.ts 对齐 */
export type EmotionKey =
  | 'happy'
  | 'calm'
  | 'excited'
  | 'anxious'
  | 'confused'
  | 'sad'
  | 'angry'
  | 'lonely';

/** 单条每日情绪记录（持久化单元） */
export interface MoodRecord {
  id: string;                 // uuid
  date: string;               // YYYY-MM-DD，每天唯一
  emotions: EmotionKey[];     // 支持单选/多选
  description: string;        // ≤300 字
  aiAnalysis: AiAnalysis | null; // 未分析或降级时为 null
  createdAt: number;          // 时间戳
}

/** 记录草稿（提交前的表单态，无 id/aiAnalysis） */
export type MoodDraft = Pick<MoodRecord, 'emotions' | 'description'>;
```

### `types/ai.ts`

```typescript
/** 情绪强度档位 */
export type EmotionIntensity = 'low' | 'medium' | 'high';

/** AI 情绪分析结果 */
export interface AiAnalysis {
  keywords: string[];          // 情绪关键词，如 ['紧张','期待','成长']
  intensity: EmotionIntensity; // 情绪强度
  feedback: string;            // 简短反馈文案
}

/** 每日陪伴语 */
export interface CompanionQuote {
  text: string;
  generatedAt: number;
}

/** AI 服务统一返回（含降级标记） */
export interface AiResult<T> {
  data: T;
  source: 'api' | 'mock';      // mock 表示降级
}
```

### `types/garden.ts`

```typescript
import type { EmotionKey } from './mood';

/** 成长阶段，由记录天数派生 */
export type GardenStage = 'seed' | 'small' | 'sea' | 'forest';
// seed: 1天 / small: 7天 / sea: 30天 / forest: 100天

/** 单种情绪的视觉配置（config/emotions.ts 的条目类型） */
export interface EmotionConfig {
  key: EmotionKey;
  label: string;               // 中文名，如 '开心'
  emoji: string;
  color: string;               // 主配色 hex
  plants: string[];            // 植物元素 id
  effects: string[];           // 粒子/特效 id
  timeOfDay: 'day' | 'dusk' | 'night' | 'overcast';
}

/** 引擎渲染所需的场景描述（由记录 + 配置派生，非持久化） */
export interface GardenScene {
  stage: GardenStage;
  emotions: EmotionKey[];      // 当天情绪
  plantCount: number;          // 随累计天数增长
  primaryColor: string;
}
```

### `context/GardenContext` 状态形状

```typescript
import type { MoodRecord, EmotionKey } from '@/types/mood';
import type { GardenStage } from '@/types/garden';

export interface GardenState {
  records: MoodRecord[];       // 全部历史记录（真相源）
  todayEmotions: EmotionKey[]; // 当天已选情绪
  stage: GardenStage;          // 派生：当前成长阶段
}

export type GardenAction =
  | { type: 'HYDRATE'; payload: MoodRecord[] }
  | { type: 'ADD_RECORD'; payload: MoodRecord }
  | { type: 'UPDATE_RECORD'; payload: MoodRecord };
```

## 八、与 CLAUDE.md 规范对照

| 规范 | 落实 |
|---|---|
| 新增功能独立 | 按领域分目录，各自闭环 |
| 禁止重构 | 引擎/AI/存储封装在边界后 |
| 优先复用 | `components/ui/` 强制复用层 |
| 组件 ≤300 行 | 细粒度拆分 + 逻辑外移 hooks |
| 复杂逻辑放 hooks | 副作用/计算集中 `hooks/` |
| 页面只展示 | `pages/` 仅布局组合 |
| 禁止 any | `types/` 集中类型 + tsconfig strict |
| 编译通过 | strict + 单一数据源降低断裂风险 |
